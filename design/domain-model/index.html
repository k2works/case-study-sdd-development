    ドメインモデル設計 - フレール・メモワール WEB ショップ¶

    ユビキタス言語¶

    用語 定義
    得意先 花束を注文する個人顧客
    届け先 花束の届け先（住所・電話番号）
    受注 得意先からの注文。1 受注 = 1 届け先 = 1 商品
    届け日 花束を届ける日
    出荷日 花束を出荷する日（= 届け日の前日）
    商品 事前定義された花束の組合せ
    商品構成 商品を構成する単品と数量
    単品 個別の花。品質維持可能日数・購入単位・リードタイムを持つ
    品質維持可能日数 入荷日から花が使用可能な日数
    購入単位 仕入先への発注時の最小単位
    発注リードタイム 発注から入荷までの日数
    仕入先 単品を供給するパートナー。単品ごとに特定
    発注 仕入先への単品の発注
    入荷 仕入先から届いた単品の受け入れ
    在庫 入荷した単品のロット。品質維持期限を持つ
    在庫推移 日別の在庫予定数（動的算出）
    結束 花材から花束を組み立てる作業
    引当 受注に対して在庫を確保すること

    集約設計¶

    集約一覧¶

    集約 ルート 含むエンティティ/VO 不変条件
    得意先 得意先 届け先 得意先は 1 つ以上の連絡先を持つ
    商品 商品 商品構成（VO） 商品は 1 つ以上の商品構成を持つ
    単品 単品 - 品質維持日数 > 0、購入単位 > 0、リードタイム >= 0
    受注 受注 届け先スナップショット（VO） 出荷日 = 届け日 - 1。1 受注 = 1 商品
    発注 発注 - 数量は購入単位の倍数。入荷予定日 = 発注日 + リードタイム
    在庫ロット 在庫ロット - 品質維持期限 = 入荷日 + 品質維持日数。引当時にロットを分割する（例: 100本ロットから3本引当 → 3本(引当済み) + 97本(有効)の2レコードに分割）

    在庫引当方針¶

    引当粒度¶

    在庫ロットは引当時に分割する。1ロットの一部を引当する場合、引当数量分の新しいロット（引当済み）と残数量のロット（有効）に分割する。

    操作 処理内容
    引当（allocate） 必要数量分のロットを分割し、引当済みにする。orderId を設定
    引当解除（deallocate） 引当済みロットを有効に戻し、orderId を null にする。同一ロットの有効分と統合可能
    消費（consume） 引当済みロットを消費済みにする

    トランザクション方針¶

    • 受注作成と在庫引当は同一トランザクションで実行する
    • 商品構成の全単品について引当を試行し、いずれかの単品で在庫不足が発生した場合は受注作成自体をロールバックする
    • 引当対象は品質維持期限が出荷日以降の有効ロットとし、入荷日が古い順（FIFO）に引当する
    • キャンセル時は該当 orderId の全引当済みロットを有効に戻す

    データモデルとの対応¶

    ドメインモデル データモデル 備考
    得意先（集約） customers + destinations 届け先は得意先集約の子エンティティ
    商品（集約） products + product_compositions 商品構成は値オブジェクトとして商品に含む
    単品 items 1:1 対応
    受注（集約） orders 届け先はスナップショットとして受注に埋め込み
    発注 purchase_orders + arrivals 入荷は発注の receive() で処理
    在庫ロット stocks ロット単位で管理
    仕入先 suppliers 単品から SupplierId で参照
