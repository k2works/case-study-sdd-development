## Keep（継続すること）

- **Service Object パターンの活用**: PurchaseOrderService に発注作成・入荷受入の両方のロジックを集約。Controller を薄く保ち、テストしやすい設計を維持
- **既存モデルの拡張による効率的な実装**: IT3 で作成済みの PurchaseOrder/Arrival/Stock モデルを活用し、新規マイグレーションなしで機能を追加
- **IT3 Try の確実な反映**: 空状態ガイダンス、共通テンプレートパターン（card + table-striped + table-dark ヘッダー）を初期実装から適用
- **トランザクション制御**: 入荷受入で Arrival 作成 + Stock 作成 + PO ステータス更新を 1 トランザクションにまとめ、データ整合性を確保

---

## Problem（問題点）

- **Date.parse の例外ハンドリング漏れ**: Controller で無効な日付文字列を受け取った場合の ArgumentError が未処理。レビューで発見・修正済み
- **ステータスフィルタのホワイトリスト未検証**: params[:status] を直接 where 句に渡していた。enum の値に限定するバリデーションが必要だった
- **エージェントレビューの API リミット**: XP エージェント 5 並列起動時に API リミットに到達。フォールバックで直接レビューを実施

---
