## KPT 分析

### Keep（継続すべきこと）

**技術的成功事項**:

- **TDD で 3 機能を安全に追加**: Order.ship()、Order.cancel()、Order.reschedule() のドメインメソッドをすべて Red-Green-Refactor で実装。境界値テスト（当日/翌日/30日後/31日後/過去日）もカバーし、DeliveryDate のバリデーションが堅牢
- **既存パターンの踏襲が効率的**: BundlingQueryService → ShipmentQueryService、BundlingController → ShipmentController のパターン再利用で、出荷機能の実装が高速。コードの一貫性も維持
- **レビュー指摘の計画段階での反映**: IT6 計画レビュー（高 10 件 + 中 4 件）を計画書に事前反映し、実装時の手戻りを最小化。特に PREPARING→CANCELLED 不許可の設計判断を計画段階で確定できた
- **deliveryDate の final 除去を安全に実行**: レビュー H-4 の指摘通り、deliveryDate フィールドを mutable に変更。全既存テストが通過し回帰なし
- **DeliveryDateChangeValidator の在庫チェック + 代替日提案**: InventoryQueryPort の 4 メソッドを再利用し、在庫充足チェックと代替日探索（最大 5 件、前後 14 日）を実装。テストでモック戦略が機能

**プロセス的成功事項**:

- **16SP を 1 セッションで完了**: 過去最大の SP（平均 11.8SP の 36% 超過）だったが、効率的な TDD サイクルとパターン再利用で完了。判断ゲートを使わずに全機能実装
- **disabled + ツールチップのパターン確立**: キャンセル不可・届け日変更不可の UI を統一パターン（disabled ボタン + title 属性でエスカレーション案内）で実装。今後の機能追加でも再利用可能
- **フロントエンドのコンポーネントテスト充実**: OrderDetailPage に 10 テスト（キャンセル 7 + 届け日変更 3）、ShipmentPage に 5 テストを追加。FE テストが 54 → 64 に増加

### Problem（問題点）

**技術的課題**:

- **Checkstyle 未使用インポートの再発**: DeliveryDateChangeValidatorTest で `anyLong` の未使用インポートが残存し、コミット時に検出。IT5 ふりかえりで同じ問題を指摘済み
- **成功基準 2 項目が未検証**: ArchUnit テストによるアーキテクチャ検証、テストカバレッジ 80% の検証が未実施。品質チェックフェーズで対応が必要
- **Clock 注入が未対応**: IT2 からの技術負債。計画にタスク 4.8 として含めたが、実装セッションでは着手できず。`Order.create()` と `DeliveryDate.validate()` が `LocalDate.now()` / `LocalDateTime.now()` を直接呼んでいる
- **StockStatus.DEGRADED 未解決**: IT4 からの持越し。ADR で方針決定とした が、IT6 でも着手できず
- **在庫引当解除のロジックが CancelOrderUseCase に未実装**: 受入条件「キャンセル時に在庫引当が解除される」は InventoryQueryPort 経由で引当情報を管理する必要があるが、現在の CancelOrderUseCase はステータス変更のみ。在庫引当の解除ロジックは Phase 2 リリース前に追加が必要

**プロセス的課題**:

- **テスト・リリース準備タスク（セクション 4）が未着手**: 統合テスト、E2E テスト、FE コンポーネントテスト拡充、Phase 2 回帰テスト、リリース準備が残存。計 18h の工数が未消化
- **フロントエンドのアイスクリームコーン傾向**: IT5 で指摘した問題の改善が限定的。ユニット:E2E = 3:1 の KPI 目標は未達成

### Try（次に試すこと）

| アクション | 担当 | 期限 | 期待効果 |
|-----------|------|------|---------|
| テスト・リリース準備タスク（4.1〜4.9）を Phase 2 リリース前に完了 | - | Release 2.0 前 | 品質ゲート達成、リリース可能状態 |
| Clock 注入を Order.create() と DeliveryDate.validate() に適用 | - | Release 2.0 前 | テスト時の日時制御が可能に |
| CancelOrderUseCase に在庫引当解除ロジックを追加 | - | Release 2.0 前 | 受入条件の完全充足 |
| StockStatus.DEGRADED の方針を ADR で決定 | - | IT7 | 不要なドメインモデル要素を排除 |
| Checkstyle の IDE 即時フィードバック設定（IT5 Try の継続） | - | IT7 | 未使用インポートの再発防止 |
