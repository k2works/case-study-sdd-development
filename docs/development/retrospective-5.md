## Problem（問題点）

| # | 項目 | 深刻度 | 詳細 | 影響 |
|---|------|--------|------|------|
| P1 | E2E テストの新規シナリオ未追加 | 中 | IT5 で S04/S02/S05 の E2E テストシナリオを追加する計画だったが、既存の 33 件のまま。得意先管理・届け先コピー・届け日変更の E2E 検証がない | 受入基準の自動検証が不完全。手動テストに依存 |
| P2 | E2E テストの本番サーバー干渉 | 低 | port 8080 で本番バックエンドが動いている状態で E2E テストを実行すると、テストデータのリセットが機能せず strict mode violation が発生 | global-setup.ts でテストモード検証を追加して対応済み |
| P3 | useApi.ts の旧 API と新 API の混在 | 低 | 旧 API（fetch ベース）と新 API（fetchApi ベース）が混在。新規追加分は fetchApi を使用しているが、既存関数は fetch を直接使用 | SonarQube カバレッジに影響。統一リファクタリングが望ましい |
| P4 | マイグレーションのタイムスタンプ順序問題 | 低 | arrivals マイグレーションのタイムスタンプが purchase_orders より前だったため、DB リセットが必要だった | 開発中に 1 回のリセットで解消。CI では問題なし |
