# イテレーション 5 ふりかえり

## 対象

| 項目 | 内容 |
|------|------|
| イテレーション | IT5 |
| 期間 | 2026-05-18 から 2026-05-29 まで |
| 対象ストーリー | `US-00`, `US-10` |
| 実績 SP | 6 / 6 |

## Keep

- `US-00` と `US-10` を同一イテレーションで完了し、商品マスタ管理から出荷確定までをつないで `Release 1.1` の業務フローを完了できた。
- 商品マスタ更新が顧客向け導線と出荷 recipe の両方へ反映されるようにし、業務データの source を product master に寄せられた。
- 実装レビューと UI/UX レビューを同日に実施し、 `新規登録` 導線と通信障害表示の不足をその場で修正できた。

## Problem

- `git-commit` を並列実行したため、レビュー文書とフロント修正が同一コミットへ混在した。
- 顧客画面の fallback 戦略はテスト耐性には有効だったが、業務上は障害可視化との両立を追加で調整する必要があった。
- `release_plan.md` と `development/index.md` の更新は依然として開発完了後の後追いになっており、終了処理の自動化余地が残っている。

## Try

| アクション | 担当 | 期限 | 期待効果 |
|-----------|------|------|----------|
| イテレーション終了時のコミットは `実装` と `レビュー文書` を逐次で分け、並列コミットを避ける | - | IT6 終了時 | 履歴の混線防止 |
| 顧客画面の通信障害方針を `障害表示 + 再試行 + 暫定表示` として明文化する | - | IT6 Day 1 | 実装と UI 設計の再ずれ防止 |
| `tracking-progress --update` と `planning-releases --report` のテンプレート化を進める | - | Release 1.2 終了時 | 終了処理の手戻り削減 |
