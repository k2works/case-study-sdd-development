## ゴール

### イテレーション終了時の達成状態

1. **発注登録**: 仕入スタッフが単品・数量・入荷予定日を入力して発注を登録できる
2. **入荷受入**: 仕入スタッフが発注番号を選択して入荷数量を登録し、在庫ロットが作成される
3. **品質期限アラート**: 品質維持期限まで残り 2 日以内の在庫ロットがアラート表示される
4. **在庫推移連携**: 発注の入荷予定が在庫推移画面に反映される

### 成功基準

- [ ] 発注登録画面で単品・数量・入荷予定日を入力して発注が作成される
- [ ] 仕入先は単品マスタから自動設定される
- [ ] 発注の入荷予定が在庫推移に反映される
- [ ] 発注番号を選択して入荷数量を登録できる
- [ ] 入荷日が品質維持日数の起算日として設定される
- [ ] 在庫ロットが作成され在庫が更新される
- [ ] 入荷数量と発注数量の差異が表示される
- [ ] 品質維持期限まで残り 2 日以内の在庫ロットがアラート表示される
- [ ] `uv run pytest` で全テストがパス
- [ ] テストカバレッジ 80% 以上（ドメイン層）

## 技術方針

### 新規 Django アプリ: purchasing

- `apps/purchasing/` を作成
- DDD レイヤー構成（domain/entities.py, domain/value_objects.py, domain/interfaces.py）
- 既存パターンに従いインサイドアウト TDD

### アーキテクチャ

- PurchaseOrder 集約と Arrival 集約を purchasing アプリに配置
- StockLot の作成は inventory アプリのサービスが担当（アプリ間連携）
- 在庫推移計算に入荷予定を反映するため StockForecastService を拡張
