## Keep（続けること）

- **TDD インサイドアウト開発**: Red-Green-Refactor サイクルによるドメイン層→アプリ層→インフラ層→API 層の順序が効果的だった。各レイヤーの責務が明確になり、テストが設計を駆動した。
- **ArchUnit による依存方向の強制**: ヘキサゴナルアーキテクチャのレイヤー間依存ルールを自動テストで検証することで、設計の劣化を早期に検出できた。
- **XP エージェントによる UI/UX レビュー**: Interaction Designer と User Representative の並列レビューにより、姓名ラベルの逆転バグ（FORM-02）を早期発見し、データ不整合を未然に防いだ。

---

## Problem（問題点）

- **Codex MCP のタイムアウト**: Codex 分業モードで計画したが、Codex MCP が毎回タイムアウトしたため Claude 単独実装にフォールバックした。計画の柔軟性が求められた。
- **テストカバレッジ 80% の確認未実施**: JaCoCo のカバレッジレポートは生成されているが、80% 以上の閾値確認を実施していない。CI での自動確認が必要。
- **レスポンシブ対応が不十分**: Interaction Designer のレビューで、テーブルのモバイル対応やヘッダーのハンバーガーメニューが未実装であることを指摘された。

---

## Try（次に試すこと）

- **Codex MCP の安定性事前確認**: IT2 では Codex MCP の接続テストを開発開始前に実施し、タイムアウトが頻発する場合は即座に Claude 単独モードに切り替える判断基準を設定する。
- **テストカバレッジの CI 自動確認**: GitHub Actions で JaCoCo カバレッジを 80% 以上の Quality Gate として設定し、PR マージ時に自動確認する。
- **モバイルファーストのレスポンシブ対応**: IT2 の技術タスクとしてレスポンシブ対応を含め、Tailwind CSS のブレークポイントを活用したモバイル対応を実装する。
