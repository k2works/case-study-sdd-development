## UI/UX レビュー結果

### レビュー対象

- 管理画面コンポーネント `apps/frontend/app/admin/page.tsx`
- 管理画面スタイル `apps/frontend/app/globals.css`
- 関連設計 `docs/design/ui-design.md`

### 総合評価

管理画面は `受注 / 在庫 / 花材 / 発注` のワークベンチ分割と 2 ペイン構成が入り、前回より設計意図に近づいています。一方で、仕入業務の主導線と状態デザインがまだ弱く、業務画面としての迷いにくさと即応性に改善余地があります。

### モダンデザイン準拠サマリー

| 評価項目 | 状態 | 備考 |
|---|---|---|
| カラーシステム | OK | 警告バッジと Surface の系統は揃っている |
| タイポグラフィ | OK | Hero / Eyebrow / Panel title の階層は分かる |
| Elevation & Surface | OK | Panel と Chip の階層は表現されている |
| コンポーネント一貫性 | 要改善 | 成功 / エラー / 再試行の扱いが画面ごとに揺れる |
| スペーシング | OK | 主要パネルは一定のリズムで並んでいる |
| レスポンシブ / Adaptive | 要改善 | モバイル時にテーブル主体画面の情報圧縮が弱い |
| ダークモード | 未対応 | ライト前提のみ |
| 状態デザイン（空 / Loading / Error） | 要改善 | Loading がなく、 Error の回復導線も一部不足 |

### 改善提案（重要度順）

#### 高（リリース前に対応すべき）

| # | 提案 | 箇所 | 指摘元 | 理由 |
|---|------|------|--------|------|
| 1 | 在庫推移から発注確定へ進む主導線を追加する | `apps/frontend/app/admin/page.tsx` の在庫管理領域 | xp-interaction-designer / xp-user-representative | 設計では `在庫推移 -> 発注作成` が主業務フローですが、実装は上部タブ切替に依存しています。業務文脈が分断され、在庫確認から次アクションへ進みにくいです。 |

#### 中（対応推奨）

| # | 提案 | 箇所 | 指摘元 | 理由 |
|---|------|------|--------|------|
| 1 | Loading 状態を各ワークベンチに追加する | `apps/frontend/app/admin/page.tsx` 全体 | xp-interaction-designer / xp-user-representative | 初期表示時は空配列と区別がつかず、「まだ読込中」なのか「本当に 0 件」なのか判断できません。 |
| 2 | 通信失敗時の再試行導線を花材管理と発注管理にも揃える | `apps/frontend/app/admin/page.tsx` の花材管理 / 発注管理 | xp-user-representative | 在庫管理だけ再試行があり、他の管理機能はエラー文のみです。業務停止時の復帰手順が一貫していません。 |
| 3 | 初期表示を役割ベースで切り替えるか、少なくとも仕入スタッフ向け入口を明示する | `apps/frontend/app/admin/page.tsx#L88` 周辺 | xp-user-representative | 設計では仕入スタッフの初期表示は `在庫管理` です。現状は常に `受注管理` から始まるため、IT3 の主利用者に不要な 1 操作を強いています。 |
| 4 | 発注結果メッセージを仕入先グループ単位で表示する | `apps/frontend/app/admin/page.tsx` の発注管理領域 | xp-interaction-designer | 複数仕入先が並ぶと、画面下部の 1 行メッセージではどの操作結果か追いにくいです。 |

#### 低（改善の余地あり）

| # | 提案 | 箇所 | 指摘元 | 理由 |
|---|------|------|--------|------|
| 1 | モバイル時のテーブル表示をカード化または横スクロール明示にする | `apps/frontend/app/globals.css` の `@media (max-width: 640px)` | xp-interaction-designer | 列数の多い管理テーブルが小画面で詰まりやすく、 Adaptive Layout の観点でまだ粗いです。 |

### 矛盾事項

なし。

### 評価サマリー

レイアウト階層、 Surface、切替 Chip は前回より改善しています。ただし、仕入業務の主要フローがタブ分断されており、状態デザインも `Loading / Action result` でまだ弱いです。

### 良い点

- `受注` と `花材` は 2 ペイン化され、一覧と編集 / 詳細の関係が読みやすい
- `status-badge` により在庫警告の視線誘導が前回より明確
- Surface とボタンスタイルは管理画面全体で概ね統一されている

### モダンデザイン準拠状況

- Elevation と Shape は一定水準で揃っている
- コンポーネントの基調色は一貫している
- ただし Adaptive Layout と状態デザインは弱い

### 改善提案

- 【重要度: 高】在庫推移から発注作成への CTA を置き、業務フローを画面内でつなぐ
- 【重要度: 中】Loading Skeleton または Loading Message を追加し、空状態と切り分ける
- 【重要度: 中】発注結果をグループごとの inline feedback にする
- 【重要度: 低】モバイル時のテーブル情報量を再設計する

### 懸念事項

- 仕入スタッフの心理モデルでは `在庫を見た次に発注する` が自然で、タブ移動は余計なモード切替になる

### スコープ外の発見

- ダークモードは未整備

<summary>xp-user-representative（高: 1 / 中: 3 / 低: 0）</summary>

### 評価サマリー

仕入に必要な機能は揃いましたが、現場利用の流れとしてはまだ「次に押す場所」が弱いです。特に通信障害時と読込中の扱いが、業務継続の観点で不足しています。

### 良い点

- 花材管理と発注管理が分かれ、役割ごとの作業対象は理解しやすい
- 花材編集フォームは項目が少なく、最低限の学習で使える
- 発注数量をその場で調整できるのは実務に合っています

### モダンデザイン準拠状況

- 他のモダンな業務 UI と比べても切替 UI とカード化は大きく崩れていない
- ただし「今読み込み中か」「失敗後にどう戻るか」が不足している

### 改善提案

- 【重要度: 高】在庫確認から発注へ直接進める導線を置く
- 【重要度: 中】花材 / 発注でも再試行ボタンを出す
- 【重要度: 中】仕入スタッフの最初の表示を `在庫管理` に寄せる
- 【重要度: 中】Loading 状態を明示する

### 懸念事項

- 回線が不安定な現場では、文面だけのエラーは「何をすれば戻るのか」が分かりにくい
- 仕入先が複数並んだとき、どの発注が成功したか一目で分かりにくい

### スコープ外の発見

- 入荷登録や出荷管理との導線は未実装のため、将来の管理ホーム設計が必要
