# UI/UX レビュー結果: IT1 View 実装

レビュー日: 2026-03-24

## レビュー対象

- `apps/app/views/` - products, items, compositions, devise, layouts

## 総合評価

フォーム設計の一貫性と構成画面のインライン UI は良好。ただし、**Bootstrap CSS が読み込まれていない**、**ナビゲーションバーが未実装**、**フラッシュメッセージの表示機構がない**という 3 つの基盤的な問題が共通指摘。IT2 開始前に対応が必要。

## 改善提案（重要度順）

### 高

| # | 提案 | 指摘元 | 理由 |
|---|------|--------|------|
| 1 | **Bootstrap CSS を読み込む**（application.scss に @import "bootstrap"） | IxD | 全画面のスタイルが無効 |
| 2 | **ナビゲーションバーを追加**（商品/単品/ログアウト） | IxD, UserRep | 画面間の遷移手段がない |
| 3 | **フラッシュメッセージの表示を追加** | IxD, UserRep | 操作結果のフィードバックがない |
| 4 | **商品一覧に「構成」リンクを追加** | IxD, UserRep | 花束構成画面への導線がない |

### 中

| # | 提案 | 指摘元 | 理由 |
|---|------|--------|------|
| 5 | 削除操作に確認ダイアログ（turbo_confirm） | IxD, UserRep | 誤削除防止 |
| 6 | 商品一覧に構成数のインジケータ追加 | UserRep | 構成未設定の商品が判別不能 |
| 7 | チェックボックスを Bootstrap form-check パターンに修正 | IxD | マークアップ不備 |
| 8 | 数値フィールドに min 属性追加 | IxD | クライアント側の入力制約なし |
| 9 | 「掲載する」のラベルを補足テキスト付きに | UserRep | 操作の意味が不明確 |

### 低

| # | 提案 | 指摘元 | 理由 |
|---|------|--------|------|
| 10 | `<html lang="ja">` 追加 | IxD | WCAG アクセシビリティ |
| 11 | テーブルを table-responsive で囲む | IxD | モバイル対応 |
| 12 | 0 件時のガイドメッセージ | UserRep | 空状態の UX |
| 13 | ページタイトルの設定 | UserRep | タブの識別 |

## 対応方針

### 修正する（IT2 開始前）

| # | 対応内容 |
|---|---------|
| 1 | Bootstrap CSS の読み込み設定 |
| 2 | レイアウトにナビゲーションバーを追加 |
| 3 | フラッシュメッセージの表示を追加 |
| 4 | 商品一覧に「構成」リンクを追加 |
| 5 | 削除ボタンに turbo_confirm を追加 |
| 10 | html タグに lang="ja" を追加 |

### 許容する（IT2 で対応）

| # | 理由 |
|---|------|
| 6-9 | IT2 の Bootstrap レイアウト本格適用時にまとめて対応 |
