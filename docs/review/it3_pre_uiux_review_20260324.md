# IT3 開発前 UI/UX レビュー統合レポート

## レビュー対象

- 実装済みテンプレート: base.html, product_list.html, product_detail.html, order_form.html, order_complete.html
- UI 設計書: docs/design/ui-design.md（画面遷移図、salt 図）
- フォーム定義: orders/forms.py

## 総合評価

IT1-IT2 で実装された花束ショップの購買フロー（一覧→詳細→注文→完了）は基本的に機能しており、ピンク系カラーパレットによるブランドメタファーの一貫性、レスポンシブグリッドの対応は評価できる。

一方で、**注文確認画面（C-07）の欠落**が最大の問題であり、US-005 の受入条件・UC-001 ステップ 7 の未充足に加え、花束という「大切な記念日に届ける贈り物」の性質上、届け先・届け日の入力ミスが取り返しのつかない事故につながる。フォームフィールド順序の設計書との乖離、アクセシビリティの基盤不足も IT3 で対応すべき。

## 改善提案（重要度順）

### 高（IT3 開発前に対応すべき）

| # | 提案 | 箇所 | 指摘元 | 理由 |
| :--- | :--- | :--- | :--- | :--- |
| UH-01 | 注文確認画面（C-07）の追加 | 画面遷移全体 | designer, user-rep | US-005 受入条件未充足。花束注文の不可逆性に対して確認ステップなしは事故リスク大 |
| UH-02 | フォームフィールド順序の修正（届け日を先頭に） | forms.py, order_form.html | designer, user-rep | UI 設計書 C-05 と乖離。ユーザーの思考フロー（いつ→どこに→何を）と不整合 |
| UH-03 | フォーカスリングの追加 | forms.py `_INPUT_CSS` | designer | WCAG 2.4.7（フォーカスの可視化）不適合。キーボード操作でフォーカス位置が不明 |
| UH-04 | 二重送信防止 | order_form.html | designer, user-rep | 連打で重複注文が発生。花束は数千円/件で金銭的損害に直結 |
| UH-05 | セマンティック HTML の改善 | 全テンプレート | designer | `<nav>` 未使用、商品カード全体が `<a>` で冗長読み上げ、チェックマークに `aria-label` なし |

### 中（IT3 中に対応推奨）

| # | 提案 | 箇所 | 指摘元 | 理由 |
| :--- | :--- | :--- | :--- | :--- |
