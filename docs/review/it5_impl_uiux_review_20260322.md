# IT5 UI/UX 実装レビュー結果

## レビュー対象

- ArrivalRegistrationPage.tsx（S-302: 入荷登録画面）
- BundlingTargetsPage.tsx（S-401: 結束対象一覧画面）
- AppLayout.tsx（ナビゲーション）
- App.tsx（ルーティング）

## 総合評価

入荷登録画面と結束対象一覧画面は基本的な業務フローを適切にカバーしており、確認ダイアログ・二重送信防止・花材不足アラート・ロールベースナビゲーションなどの堅実な実装がされています。しかし、UI 設計ドキュメントで定義された複数の重要な仕様（入荷数量のデフォルト値、トースト通知、届け先列、アコーディオン展開、タッチターゲットサイズ）が未実装であり、連続作業の効率性・アクセシビリティ・モダンデザイン準拠の観点で改善が必要です。

## モダンデザイン準拠サマリー

| 評価項目 | 状態 | 備考 |
|---|---|---|
| カラーシステム | OK | emerald ベースの一貫したカラースキーム |
| タイポグラフィ | 要改善 | ページタイトルの font-semibold / font-bold 不統一 |
| Elevation & Surface | OK | カード・テーブルのシャドウが適切 |
| コンポーネント一貫性 | 要改善 | ダイアログの ARIA 属性が画面間で不統一、ボタンサイズ不統一 |
| スペーシング | OK | 8px グリッドに概ね準拠 |
| レスポンシブ / Adaptive | 要改善 | モバイル・タブレット対応なし、ナビ溢れリスク |
| ダークモード | 未対応 | dark: プレフィックス未使用 |
| 状態デザイン（空 / Loading / Error） | OK | スケルトン・エラー・空状態が実装済み |
| デザイントークン | 要改善 | CSS 変数未使用、Tailwind ハードコーディング |

## 改善提案（重要度順）

### 高（リリース前に対応すべき）

| # | 提案 | 箇所 | 指摘元 | 理由 |
|---|------|------|--------|------|
| H-1 | 入荷数量のデフォルト値を残数量にする | ArrivalRegistrationPage L18 | designer, userrep | 設計仕様（UI/UX-H-4）。全量入荷が8割以上で毎回0から入力は非効率 |
| H-2 | 成功後のトースト通知を実装する | ArrivalRegistrationPage L51-53 | designer, userrep | 設計仕様（UI/UX-H-1）。連続作業で「登録できたか」のフィードバックがない |
| H-3 | 発注日・希望納品日・既入荷数量・ステータスを表示する | ArrivalRegistrationPage L108-125 | designer, userrep | 設計仕様（UI/UX-M-4）。遅延入荷の把握に必要 |
| H-4 | 結束対象一覧に「届け先」列を追加する | BundlingTargetsPage L83-89 | designer, userrep | 設計仕様。花束の取り違え防止に必須 |
| H-5 | BundlingTargetsPage の確認ダイアログに ARIA 属性を追加 | BundlingTargetsPage L165 | designer | `role="dialog"` `aria-modal` `aria-labelledby` が欠如。ArrivalRegistrationPage と不統一 |
| H-6 | 花材サマリの不足数に aria-label とアイコンを追加 | BundlingTargetsPage L151-155 | designer, userrep | 設計仕様（UI/UX-H-2）。色のみ依存は WCAG 1.4.1 違反 |
| H-7 | 残数量超過のフロントエンドバリデーションを追加 | ArrivalRegistrationPage handleSubmit | designer, userrep | HTML max 属性のみでは不十分。具体的な数値を含むエラーメッセージが必要 |

### 評価サマリー
入荷登録画面・結束対象一覧画面ともに基本的な業務フローは実現できており「使えない」レベルではない。しかし、設計ドキュメントで明確に定義された UI/UX 仕様のうち、業務効率と安全性に直結する複数の要素が実装から漏れており、このまま現場に出すと仕入スタッフの連続作業効率が落ち、フローリストが結束作業中に必要な情報を得られない場面が生じる。

### 主要な指摘
- H-1: 入荷数量のデフォルト値が 0（設計は残数量）
- H-2: 成功後のトースト通知がない
- H-3: 残数量超過のフロントエンドバリデーション不十分
- H-4: 届け先が表示されていない
- H-5: 花材構成のアコーディオン展開が未実装
- M-1: 発注日・希望納品日が表示されていない
- M-2: 結束完了ボタンのタッチターゲットが小さい（48px 未満）
- M-3: 結束完了後のボタンが「完了済み」ではなく非表示
- M-4: 花材不足時の結束完了ダイアログに強い警告がない
- M-5: 見出しに「準備日」の表示がない
- L-1: 空状態メッセージが設計と微妙に異なる
- L-2: ナビラベル「結束管理」がフローリストに馴染みにくい
- L-3: 入荷数量フィールドの全選択が未実装

### 懸念事項
- 連続入荷登録のワークフロー効率（1件あたりの操作ステップが多い）
- 結束作業中の画面更新で行位置がずれるリスク
- ルーティングのロール制御不足（URL 直接入力でアクセス可能）
