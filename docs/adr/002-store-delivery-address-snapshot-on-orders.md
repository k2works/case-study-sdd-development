# ADR-002: 受注に届け先スナップショットを保持する

過去受注の事実を保全するため、受注は届け先履歴への参照に加えて、注文時点の届け先スナップショットを保持します。

日付: 2026-03-19

## ステータス

承認済み

## コンテキスト

- 顧客は過去の届け先を再利用できるが、届け先履歴は後日更新される可能性がある
- 受注確認、出荷実績、問い合わせ対応では、注文時点の届け先を正確に再現できる必要がある
- `delivery_addresses` のみを参照すると、履歴編集によって過去受注の表示内容が変化してしまう
- レビューでも、監査性とテスト安定性の観点からスナップショット保持が必要と指摘された

## 決定

`orders` に、`delivery_address_id` による履歴参照を残しつつ、受注時点の受取人名、郵便番号、住所、電話番号をスナップショットとして保持する。

### 代替案

- 届け先履歴のみを参照する
  - 却下理由: 履歴更新時に過去受注の事実が壊れる
- 完全に別の `order_delivery_addresses` テーブルを持つ
  - 却下理由: 初期段階では `orders` へのスナップショット列追加で十分であり、過剰分割になる

## 影響

### ポジティブ

- 過去受注の表示と監査性を保てる
- 受け入れテストと問い合わせ対応の前提が安定する
- 履歴再利用と事実保持を両立できる

### ネガティブ

- 住所情報の重複保存が発生する
- 個人情報の保持量が増えるため、非機能要件と運用要件で保護方針を明確にする必要がある

## コンプライアンス

- 受注確定時にスナップショット列が必ず保存されること
- 受注参照画面と出荷画面がスナップショットを基準に表示できること
- 届け先履歴の更新が過去受注に影響しないことをテストで確認すること
