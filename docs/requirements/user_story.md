## ユーザーストーリー詳細

### US-01: 商品一覧を見る

**得意先として、花束の商品一覧を見たい。なぜなら、どんな花束があるかを確認して注文したいからだ。**

対応 UC: UC-01

受入条件:

- [ ] 商品名が一覧表示されること
- [ ] 商品の価格が表示されること
- [ ] 商品を選択して注文画面に進めること

---

### US-02: 花束を注文する

**得意先として、花束を注文したい。なぜなら、大切な記念日に指定した場所へ花束を届けてもらいたいからだ。**

対応 UC: UC-02

受入条件:

- [ ] 商品を選択できること
- [ ] 届け日（翌々日以降）を指定できること
- [ ] 届け先（氏名・住所）を入力できること
- [ ] メッセージを入力できること
- [ ] 注文を確定すると受注が登録されること
- [ ] 在庫不足の場合はエラーが表示されること

---

### US-03: 過去の届け先をコピーする

**得意先として、過去に使った届け先をコピーしたい。なぜなら、毎回同じ情報を入力する手間を省きたいからだ。**

対応 UC: UC-03

受入条件:

- [ ] 過去の注文の届け先が一覧表示されること
- [ ] 届け先を選択すると入力フォームに反映されること

---

### US-04: 届け日を変更する

**得意先として、注文後に届け日を変更したい。なぜなら、都合が変わった場合に対応してもらいたいからだ。**

対応 UC: UC-04

受入条件:

- [ ] 出荷前の受注の届け日を変更できること
- [ ] 変更後の届け日に在庫がある場合は変更が確定されること
- [ ] 在庫不足の場合は変更不可が通知されること

---

### US-05: 受注一覧を確認する

**受注スタッフとして、受注一覧を確認したい。なぜなら、今日・明日の出荷準備を把握したいからだ。**

対応 UC: UC-05

受入条件:

- [ ] 受注一覧が届け日順に表示されること
- [ ] 受注ステータス（受注済・出荷済）が表示されること
- [ ] 受注件数が確認できること

---

### US-06: 受注詳細を確認する

**受注スタッフとして、受注の詳細を確認したい。なぜなら、届け先・メッセージ・商品内容を正確に把握したいからだ。**

対応 UC: UC-06

受入条件:

- [ ] 商品名・届け日・届け先・メッセージが表示されること
- [ ] 花束の構成単品と数量が表示されること

---

### US-07: 在庫推移を確認する

**仕入スタッフとして、単品ごとの日別在庫推移を確認したい。なぜなら、発注が必要な単品とタイミングを判断したいからだ。**

対応 UC: UC-07

受入条件:

- [ ] 単品ごとの日別在庫予定数が表示されること
- [ ] 品質維持日数を超える在庫が識別できること
- [ ] 受注による引当数が反映されること
- [ ] 発注・入荷予定が反映されること

---

### US-08: 仕入先に発注する

**仕入スタッフとして、単品を仕入先に発注したい。なぜなら、在庫不足を事前に解消したいからだ。**

対応 UC: UC-08

受入条件:

- [ ] 単品と発注数量を入力して発注できること
- [ ] 発注情報が登録されること
- [ ] 在庫推移に発注予定数が反映されること

---

### US-09: 入荷を登録する

**仕入スタッフとして、仕入先から届いた単品の入荷を登録したい。なぜなら、在庫数を正確に把握したいからだ。**

対応 UC: UC-09

受入条件:

- [ ] 発注に対して入荷数量を登録できること
- [ ] 入荷登録後に在庫数が更新されること
- [ ] 入荷日が記録されること（品質維持日数の起算日）

---

### US-10: 出荷対象を確認する

**フローリスト・受注スタッフとして、当日の出荷対象を確認したい。なぜなら、結束に必要な花材と数量を把握したいからだ。**

対応 UC: UC-10

受入条件:

- [ ] 当日出荷対象（届け日が翌日）の受注が一覧表示されること
- [ ] 各受注の花束構成（単品名・数量）が確認できること
- [ ] 単品ごとの合計必要数が確認できること

---

### US-11: 出荷を登録する

**受注スタッフとして、花束の出荷を登録したい。なぜなら、出荷済みの受注を管理したいからだ。**

対応 UC: UC-11

受入条件:

- [ ] 出荷対象の受注を選択して出荷登録できること
- [ ] 出荷登録後に受注ステータスが出荷済に更新されること
- [ ] 出荷日が記録されること

---

### US-12: 商品マスタを管理する

**受注スタッフとして、花束の商品マスタを管理したい。なぜなら、販売する花束の構成を正確に登録・更新したいからだ。**

対応 UC: UC-12

受入条件:

- [ ] 商品（花束）を新規登録できること
- [ ] 商品の構成単品と数量を登録・更新できること
- [ ] 商品を無効化できること

---

### US-13: 単品マスタを管理する

**仕入スタッフとして、単品のマスタを管理したい。なぜなら、在庫推移計算に必要な品質維持日数・購入単位・リードタイムを正確に管理したいからだ。**

対応 UC: UC-13

受入条件:

- [ ] 単品を新規登録できること
- [ ] 品質維持日数・購入単位・リードタイム・仕入先を登録・更新できること

---

### US-14: 得意先を管理する

**受注スタッフとして、得意先情報を管理したい。なぜなら、顧客の注文履歴と届け先情報を把握したいからだ。**

対応 UC: UC-14

受入条件:

- [ ] 得意先情報（氏名・連絡先）を登録・更新できること
- [ ] 得意先の注文履歴が確認できること
