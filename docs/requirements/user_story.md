# ユーザーストーリー - フレール・メモワール WEB ショップシステム

## フェーズ 1: MVP

### 商品管理

#### US-001: 単品マスタを登録する

**として**: 受注スタッフ
**したい**: 単品（花）の情報（名前・品質維持日数・購入単位・リードタイム・仕入先）を登録したい
**なぜなら**: 商品構成と在庫管理の基盤データが必要だからだ

**受け入れ基準**:

- [ ] 単品の名前、品質維持日数、購入単位、リードタイム、仕入先を登録できる
- [ ] 登録済みの単品を更新できる
- [ ] 単品一覧を表示できる

**UC との対応**: UC-010

**ストーリーポイント**: 3

---

#### US-002: 商品（花束）を登録する

**として**: 受注スタッフ
**したい**: 商品（花束）の情報（名前・価格・説明）を登録したい
**なぜなら**: 得意先が WEB ショップで注文する商品の基盤データが必要だからだ

**受け入れ基準**:

- [ ] 商品の名前、価格、説明を登録できる
- [ ] 登録済みの商品を更新できる
- [ ] 商品一覧を表示できる

**UC との対応**: UC-010

**ストーリーポイント**: 3

---

#### US-003: 花束構成を定義する

**として**: 受注スタッフ
**したい**: 花束を構成する単品と数量を定義したい
**なぜなら**: 在庫引当の計算と結束作業に必要だからだ

**受け入れ基準**:

- [ ] 商品に対して単品と数量の組み合わせを定義できる
- [ ] 花束構成を更新できる
- [ ] 商品詳細画面で構成花材を確認できる

**UC との対応**: UC-010

**ストーリーポイント**: 3

---

### WEB 受注

#### US-004: 商品を選択する

**として**: 得意先
**したい**: WEB ショップで商品（花束）を選び、詳細（構成花材・価格）を確認したい
**なぜなら**: 贈る花束を比較検討して選びたいからだ

**受け入れ基準**:

- [ ] 商品一覧が表示される
- [ ] 商品の詳細（構成花材・価格・説明）を確認できる
- [ ] 商品を選択して注文入力に進める

**UC との対応**: UC-001 ステップ 1-2

**ストーリーポイント**: 3

---

#### US-005: 届け日・届け先・メッセージを入力して注文する

**として**: 得意先
**したい**: 届け日・届け先・メッセージカードの内容を入力して注文を確定したい
**なぜなら**: 大切な記念日に花束を届けたいからだ

**受け入れ基準**:

- [ ] 届け日を選択できる（最短: 注文日 + リードタイム日数後、最長: 注文日 + 30 日後）
- [ ] 届け先情報（氏名・住所・電話番号）を入力できる
- [ ] メッセージカードの内容を入力できる
- [ ] 注文内容の確認画面が表示される
- [ ] 注文を確定すると受注が登録される
- [ ] 注文確認メールが送信される

**UC との対応**: UC-001 ステップ 3-9

**ストーリーポイント**: 5

---

#### US-006: 届け先をコピーして再注文する

**として**: 得意先（リピーター）
**したい**: 過去の注文から届け先情報をコピーして注文したい
**なぜなら**: 住所の再入力が面倒で、入力ミスも防ぎたいからだ

**受け入れ基準**:

- [ ] 注文入力画面で「過去の届け先を利用」を選択できる
- [ ] 過去の届け先一覧が表示される
- [ ] 選択した届け先が注文入力画面にコピーされる
- [ ] コピーした届け先を修正できる

**UC との対応**: UC-002

**ストーリーポイント**: 3

---

### 在庫管理

#### US-007: 在庫推移を確認する

**として**: 仕入スタッフ
**したい**: 単品ごとの日別在庫推移（在庫残・出庫予定・入荷予定・廃棄予定）を確認したい
**なぜなら**: 適切な発注判断を行い、廃棄ロスを削減したいからだ

**受け入れ基準**:

- [ ] 単品ごとの日別在庫推移が表示される
- [ ] 在庫残、出庫予定、入荷予定、廃棄予定が表示される
- [ ] 受注・入荷登録時にイベント駆動で即時更新される

**UC との対応**: UC-007

**ストーリーポイント**: 8

---

## フェーズ 2: 仕入・出荷管理

### 仕入管理

#### US-008: 仕入先に発注する

**として**: 仕入スタッフ
**したい**: 不足する花材を仕入先に発注登録したい
**なぜなら**: 必要な花材を必要なタイミングで確保したいからだ

**受け入れ基準**:

- [ ] 単品・数量・入荷予定日を入力して発注を登録できる
- [ ] 仕入先は単品マスタから自動設定される
- [ ] 発注が入荷予定として在庫推移に反映される

**UC との対応**: UC-008

**ストーリーポイント**: 5

---

#### US-009: 入荷を受け入れる

**として**: 仕入スタッフ
**したい**: 仕入先から届いた花材を在庫に登録したい
**なぜなら**: 在庫推移を正確に把握したいからだ

**受け入れ基準**:

- [ ] 発注番号を選択して入荷数量を登録できる
- [ ] 入荷日が品質維持日数の起算日として設定される
- [ ] 在庫ロットが作成され、在庫が更新される
- [ ] 入荷数量が発注数量と異なる場合に差異が表示される

**UC との対応**: UC-009

**ストーリーポイント**: 5

---

### 出荷管理

#### US-010: 結束対象を確認する

**として**: フローリスト
**したい**: 出荷日当日に結束すべき花束と必要な花材を確認したい
**なぜなら**: 結束作業に必要な準備をしたいからだ

**受け入れ基準**:

- [ ] 出荷日（= 本日）の結束対象一覧が表示される
- [ ] 各花束に必要な花材と数量が表示される

**UC との対応**: UC-011

**ストーリーポイント**: 3

---

#### US-011: 出荷処理を行い通知する

**として**: 受注スタッフ
**したい**: 結束済みの花束を出荷処理し、得意先に出荷通知を送りたい
**なぜなら**: 確実に届けて得意先に安心してもらいたいからだ

**受け入れ基準**:

- [ ] 出荷対象の受注を確認し、出荷処理を実行できる
- [ ] 受注ステータスが「出荷済み」に更新される
- [ ] 出荷通知メールが得意先に送信される

**UC との対応**: UC-006

**ストーリーポイント**: 5

---

### 品質管理

#### US-012: 品質維持期限アラートを確認する

**として**: 仕入スタッフ
**したい**: 品質維持期限が近い（残り 2 日以内）在庫ロットを把握したい
**なぜなら**: 廃棄前に優先的に活用する判断をしたいからだ

**受け入れ基準**:

- [ ] 品質維持期限まで残り 2 日以内の在庫ロットがアラート表示される
- [ ] 在庫ロットごとに単品名、残数量、品質維持期限日が表示される

**UC との対応**: UC-007（品質維持期限確認部分）

**ストーリーポイント**: 3

---

## フェーズ 3: 顧客管理の拡充

### 届け日変更・キャンセル

#### US-013: 届け日を変更する

**として**: 得意先
**したい**: 注文後に届け日を変更したい
**なぜなら**: 予定が変わることがあるからだ

**受け入れ基準**:

- [ ] 届け日の 3 日前（= 出荷日の 2 日前）まで変更できる
- [ ] 変更先の届け日に在庫が確保可能か確認される
- [ ] 在庫引当が再計算される
- [ ] 変更不可の場合は代替日が提案される
- [ ] 変更期限超過の場合はエラーメッセージが表示される

**UC との対応**: UC-003

**ストーリーポイント**: 5

---

#### US-014: 注文をキャンセルする

**として**: 得意先
**したい**: 注文をキャンセルしたい
**なぜなら**: 事情が変わり花束が不要になることがあるからだ

**受け入れ基準**:

- [ ] 届け日の 3 日前（= 出荷日の 2 日前）までキャンセルできる
- [ ] 在庫引当が解除される
- [ ] 受注ステータスが「キャンセル済み」に更新される
- [ ] キャンセル期限超過の場合はエラーメッセージが表示される

**UC との対応**: UC-004

**ストーリーポイント**: 3

---

### 受注管理

#### US-015: 受注状況を確認する

**として**: 受注スタッフ
**したい**: 受注の一覧とステータスを確認したい
**なぜなら**: 対応が必要な受注を把握して業務を効率化したいからだ

**受け入れ基準**:

- [ ] 受注一覧が表示される
- [ ] ステータス（受付済み・出荷準備中・出荷済み・キャンセル済み）で絞り込める
- [ ] 日付範囲で絞り込める
- [ ] 個別の受注詳細を確認できる

**UC との対応**: UC-005

**ストーリーポイント**: 3
