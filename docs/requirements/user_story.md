## ストーリー詳細

### S01: 商品を登録する

**として**: スタッフ
**したい**: 商品（花束）の名前・説明・価格を登録したい
**なぜなら**: WEB ショップに商品を掲載し、得意先が注文できるようにするため

**受け入れ基準**:

- [ ] 商品名・説明・価格を入力して商品を登録できる
- [ ] 登録した商品が商品一覧に表示される
- [ ] 商品名が未入力の場合はエラーが表示される
- [ ] 価格が 0 以下の場合はエラーが表示される

---

### S02: 単品を管理する

**として**: スタッフ
**したい**: 単品（花）の名前・品質維持日数・購入単位・リードタイム・仕入先を登録・更新したい
**なぜなら**: 在庫推移の計算に必要な正確なマスタデータを管理するため

**受け入れ基準**:

- [ ] 単品名・品質維持日数・購入単位・リードタイムを入力して登録できる
- [ ] 単品に仕入先を紐づけることができる
- [ ] 登録した単品が単品一覧に表示される
- [ ] 品質維持日数が 0 以下の場合はエラーが表示される
- [ ] 購入単位が 0 以下の場合はエラーが表示される

---

### S03: 花束構成を定義する

**として**: スタッフ
**したい**: 商品（花束）を構成する単品と数量を定義したい
**なぜなら**: 結束作業に必要な情報を提供し、在庫引当の計算を正確にするため

**受け入れ基準**:

- [ ] 商品を選択して構成する単品と数量を追加できる
- [ ] 構成を変更・削除できる
- [ ] 数量が 0 以下の場合はエラーが表示される
- [ ] 未登録の単品は構成に追加できない

---

### S04a: 商品を選択する

**として**: 得意先
**したい**: 商品カタログから花束を選択して注文を開始したい
**なぜなら**: 記念日に大切な人に花束を届けたいため

**受け入れ基準**:

- [ ] 商品カタログに登録済みの花束が一覧表示される
- [ ] 花束の名前・説明・価格が表示される
- [ ] 花束を選択して注文入力画面に進むことができる

---

### S04b: 注文を確定する

**として**: 得意先
**したい**: 届け日・届け先・メッセージを入力して注文を確定したい
**なぜなら**: 届け先に確実に届けてほしいため

**受け入れ基準**:

- [ ] 届け日を指定できる（注文日の翌日以降）
- [ ] 届け先（氏名・住所・電話番号）を入力できる
- [ ] お届けメッセージを入力できる
- [ ] 注文確認画面で入力内容を確認できる
- [ ] 注文を確定すると受注が登録される
- [ ] 届け日が過去日の場合はエラーが表示される

---

### S05: 届け日を変更する

**として**: スタッフ
**したい**: 得意先の依頼に基づき、注文済みの花束の届け日を変更したい
**なぜなら**: 顧客の都合変更に迅速に対応するため

**受け入れ基準**:

- [ ] 受注済みの注文に対して届け日を変更できる
- [ ] 変更後の届け日で在庫が確保できる場合のみ変更が成功する
- [ ] 在庫が確保できない場合は変更不可の理由が表示される
- [ ] 出荷済みの注文は届け日を変更できない

---

### S06: 届け先をコピーする

**として**: 得意先
**したい**: 過去の注文の届け先をコピーして新しい注文に利用したい
**なぜなら**: 同じ届け先に何度も送る場合に入力の手間を省きたいため

**受け入れ基準**:

- [ ] 過去の注文から届け先の一覧が表示される
- [ ] 選択した届け先の情報（氏名・住所・電話番号）が注文入力画面に反映される
- [ ] 過去の注文がない場合は一覧が空で表示される

---

### S07: 受注を確認する

**として**: スタッフ
**したい**: 受注一覧を確認し、条件で絞り込みたい
**なぜなら**: 注文状況を効率的に把握し、対応が必要な注文を特定するため

**受け入れ基準**:

- [ ] 受注一覧が表示される
- [ ] 届け日で絞り込みができる
- [ ] 受注状態（受注済み・出荷済み等）で絞り込みができる
- [ ] 受注の詳細（商品・届け先・届け日等）が確認できる

---

### S08: 在庫推移を確認する

**として**: スタッフ
**したい**: 単品ごとの日別在庫推移を確認したい
**なぜなら**: 在庫の過不足を事前に把握し、廃棄ロスを最小化するため

**受け入れ基準**:

- [ ] 単品ごとの日別在庫推移が表示される
- [ ] 良品在庫・入荷予定・引当済み・廃棄対象が区分して表示される
- [ ] 品質維持日数を考慮した推移が計算される
- [ ] 特定の単品を選択して詳細を確認できる

---

### S09: 発注する

**として**: スタッフ
**したい**: 仕入先に単品を発注したい
**なぜなら**: 在庫が不足する前に必要な花材を確保するため

**受け入れ基準**:

- [ ] 単品を選択して発注数量と希望納品日を入力できる
- [ ] 仕入先・購入単位・リードタイムが自動表示される
- [ ] 発注を確定すると発注が記録され、入荷予定に反映される
- [ ] 発注数量が購入単位の整数倍でない場合はエラーが表示される

---

### S10: 入荷を受け入れる

**として**: スタッフ
**したい**: 仕入先から届いた単品の入荷を受け入れたい
**なぜなら**: 在庫推移を最新の状態に保つため

**受け入れ基準**:

- [ ] 発注一覧から対象の発注を選択できる
- [ ] 入荷数量を入力して入荷を記録できる
- [ ] 入荷を記録すると在庫が増加する
- [ ] 発注が「入荷済み」に更新される

---

### S11: 出荷一覧を確認する

**として**: スタッフ
**したい**: 出荷日に基づく出荷対象の受注を一覧で確認したい
**なぜなら**: 結束が必要な花束と出荷対象を漏れなく把握するため

**受け入れ基準**:

- [ ] 出荷日（届け日の前日）に基づく出荷対象の受注が一覧表示される
- [ ] 各受注の花束構成（単品・数量）が確認できる
- [ ] 出荷日で絞り込みができる

---

### S12: 出荷処理を行う

**として**: スタッフ
**したい**: 結束済みの花束の出荷処理を行いたい
**なぜなら**: 出荷済みの受注を正確に記録するため

**受け入れ基準**:

- [ ] 出荷一覧から出荷対象の受注を選択できる
- [ ] 出荷処理を実行すると受注状態が「出荷済み」に更新される
- [ ] 出荷処理により在庫が消費される
- [ ] 既に出荷済みの受注には出荷処理を実行できない

---

### S13: 得意先を管理する

**として**: スタッフ
**したい**: 得意先の氏名・連絡先を登録・更新したい
**なぜなら**: 得意先との取引履歴を管理し、リピーター対応を改善するため

**受け入れ基準**:

- [ ] 得意先の氏名・連絡先を入力して登録できる
- [ ] 登録した得意先が得意先一覧に表示される
- [ ] 得意先の情報を更新できる
- [ ] 氏名が未入力の場合はエラーが表示される

---

### S14: 注文をキャンセルする

**として**: 得意先
**したい**: 注文済みの花束の注文をキャンセルしたい
**なぜなら**: 不要になった注文を取り消したいため

**受け入れ基準**:

- [ ] 受注済みの注文に対してキャンセルを依頼できる
- [ ] キャンセルを実行すると受注状態が「キャンセル」に更新される
- [ ] キャンセルにより在庫引当が解除される
- [ ] 出荷済みの注文はキャンセルできない

---
