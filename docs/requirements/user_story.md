# ユーザーストーリー - フレール・メモワール WEB ショップシステム

## ストーリー一覧

### 商品管理

#### US-001: 商品（花束）を登録する

**として**: 経営者
**したい**: 新しい花束（商品）をシステムに登録したい
**なぜなら**: WEB ショップで販売する商品ラインナップを整備するためだ

**受け入れ基準**:

- [ ] 商品名、価格、説明を入力して花束を登録できる
- [ ] 登録した花束が商品一覧に表示される
- [ ] 必須項目（商品名、価格）が未入力の場合はエラーが表示される

**対応ユースケース**: UC-009

---

#### US-002: 花束の構成を定義する

**として**: 経営者
**したい**: 花束に使用する単品（花材）とその数量を定義したい
**なぜなら**: 花束の構成が在庫管理と結束作業の基盤となるためだ

**受け入れ基準**:

- [ ] 花束に対して単品と数量の組合せを登録できる
- [ ] 1 つの花束に複数の単品を紐づけられる
- [ ] 構成を変更すると在庫推移の計算に反映される

**対応ユースケース**: UC-009

---

#### US-003: 単品（花材）を登録する

**として**: 経営者
**したい**: 単品（花材）の基本情報をシステムに登録したい
**なぜなら**: 品質維持日数やリードタイムが在庫管理の判断基盤となるためだ

**受け入れ基準**:

- [ ] 単品名、品質維持日数、購入単位、リードタイム、仕入先を入力して登録できる
- [ ] 登録した単品が単品一覧に表示される
- [ ] 必須項目が未入力の場合はエラーが表示される

**対応ユースケース**: UC-010

---

### 受注管理

#### US-004: 商品一覧を表示する

**として**: 得意先
**したい**: WEB ショップで販売中の花束の一覧を見たい
**なぜなら**: 贈りたい花束を選ぶためだ

**受け入れ基準**:

- [ ] 商品一覧に花束の名前、価格、説明が表示される
- [ ] 商品を選択すると詳細情報（構成する花材等）が表示される

**対応ユースケース**: UC-001

---

#### US-005: 花束を注文する

**として**: 得意先
**したい**: 花束を選択し、届け日・届け先・メッセージを指定して注文したい
**なぜなら**: 大切な記念日に花束を届けたいためだ

**受け入れ基準**:

- [ ] 商品を選択し、届け日・届け先・メッセージを入力して注文できる
- [ ] 注文確認画面で注文内容を確認できる
- [ ] 注文確定後、受注が「注文受付」ステータスで登録される
- [ ] 届け日は翌日〜30 日後の範囲で指定できる
- [ ] 届け先の必須項目（氏名、住所）が未入力の場合はエラーが表示される
- [ ] お届けメッセージは最大 200 文字まで入力できる

**対応ユースケース**: UC-001

---

#### US-006: 受注一覧を確認する

**として**: 受注スタッフ
**したい**: 受注の一覧をステータス別に確認したい
**なぜなら**: 日々の受注状況を把握し、漏れなく処理するためだ

**受け入れ基準**:

- [ ] 受注一覧にステータス、商品名、届け日、得意先が表示される
- [ ] ステータスや期間で絞り込みができる
- [ ] 受注を選択すると詳細情報が表示される

**対応ユースケース**: UC-003

---

#### US-007: 受注を受け付ける

**として**: 受注スタッフ
**したい**: 「注文受付」の受注を確認し、「受付済み」に更新したい
**なぜなら**: 受注を確認済みとしてマークし、後続の処理に進めるためだ

**受け入れ基準**:

- [ ] 受注詳細画面から「受付済み」にステータスを更新できる
- [ ] ステータス更新後、受注一覧に反映される

**対応ユースケース**: UC-003

---

#### US-008: 届け日を変更する

**として**: 受注スタッフ
**したい**: 得意先の依頼に基づいて届け日を変更したい
**なぜなら**: 得意先の都合に合わせて柔軟に対応するためだ

**受け入れ基準**:

- [ ] 受注詳細画面から新しい届け日を入力できる
- [ ] システムが在庫推移を確認し、変更の可否を判定する
- [ ] 変更可能な場合、届け日が更新される
- [ ] 変更不可の場合、在庫不足の理由と代替日が表示される
- [ ] 「出荷準備中」以降のステータスでは変更できない

**対応ユースケース**: UC-002

---

### 在庫・仕入管理

#### US-009: 在庫推移を表示する

**として**: 仕入スタッフ
**したい**: 単品ごとの日別在庫予定数を確認したい
**なぜなら**: 品質維持日数を考慮した適切な発注判断をするためだ

**受け入れ基準**:

- [ ] 単品ごとの日別在庫予定数が表示される
- [ ] 品質維持日数に基づく廃棄予定が表示される
- [ ] 入荷予定・受注引当・廃棄予定の内訳が確認できる
- [ ] 特定の単品に絞り込み表示ができる

**対応ユースケース**: UC-004

---

#### US-010: 単品を発注する

**として**: 仕入スタッフ
**したい**: 在庫推移に基づいて不足する単品を仕入先に発注したい
**なぜなら**: 必要な花材を適切なタイミングで確保するためだ

**受け入れ基準**:

- [ ] 発注する単品を選択すると、仕入先・購入単位・リードタイムが表示される
- [ ] 発注数量と希望納品日を入力して発注できる
- [ ] 発注が「発注済み」ステータスで登録される
- [ ] 在庫推移に入荷予定として反映される

**対応ユースケース**: UC-005

---

#### US-011: 入荷を登録する

**として**: 仕入スタッフ
**したい**: 仕入先から届いた単品の入荷実績を登録したい
**なぜなら**: 在庫数量を最新の状態に保つためだ

**受け入れ基準**:

- [ ] 発注一覧から対象の発注を選択して入荷登録できる
- [ ] 入荷数量を入力すると在庫が更新される
- [ ] 全量入荷の場合、発注ステータスが「入荷済み」に更新される
- [ ] 一部入荷の場合、発注ステータスが「一部入荷」に更新される

**対応ユースケース**: UC-006

---

### 出荷・配送管理

#### US-012: 結束対象を確認する

**として**: フローリスト
**したい**: 本日の結束対象の受注と必要な花材の一覧を確認したい
**なぜなら**: 効率的に結束作業を進めるためだ

**受け入れ基準**:

- [ ] 本日出荷予定（届け日が翌日）の受注一覧が表示される
- [ ] 各受注に必要な花材と数量が表示される
- [ ] FLORIST / DELIVERY_STAFF ロールでアクセスできる

**対応ユースケース**: UC-007

---

#### US-013: 結束完了を登録する

**として**: フローリスト
**したい**: 花束の結束が完了したことを登録したい
**なぜなら**: 出荷準備が完了したことを配送スタッフに伝えるためだ

**受け入れ基準**:

- [ ] 結束対象の受注を選択して結束完了を登録できる
- [ ] 在庫から花材が消費される
- [ ] 受注ステータスが「出荷準備中」に更新される

**対応ユースケース**: UC-007

---

#### US-014: 出荷処理を実行する

**として**: 配送スタッフ
**したい**: 出荷対象の花束を出荷処理したい
**なぜなら**: 届け日に確実に届けるためだ

**受け入れ基準**:

- [ ] 本日の出荷対象一覧（「出荷準備中」の受注）が表示される
- [ ] 出荷処理を実行すると受注ステータスが「出荷済み」に更新される
- [ ] 届け先情報が配送情報として確認できる

**対応ユースケース**: UC-008

---

### 顧客管理

#### US-015: 届け先をコピーする

**として**: 得意先
**したい**: 過去の注文で使用した届け先情報を再利用したい
**なぜなら**: リピート注文の際に同じ情報を再入力する手間を省きたいためだ

**受け入れ基準**:

- [ ] 注文画面の届け先入力で「過去の届け先から選択」が選べる
- [ ] 過去の届け先が一覧表示される
- [ ] 選択すると届け先情報（氏名、住所、電話番号）が注文画面にコピーされる
- [ ] コピー後に情報を編集できる

**対応ユースケース**: UC-011

---

#### US-016: 得意先情報を確認する

**として**: 経営者
**したい**: 得意先の基本情報と注文履歴を確認したい
**なぜなら**: リピーターの状況を把握し、サービス改善に活用するためだ

**受け入れ基準**:

- [ ] 得意先一覧から得意先を検索・選択できる
- [ ] 得意先の基本情報（氏名、連絡先等）が表示される
- [ ] 得意先の過去の注文履歴が表示される

**対応ユースケース**: UC-012

---

### 認証

#### US-017: システムにログインする

**として**: システム利用者（得意先・スタッフ）
**したい**: メールアドレスとパスワードでシステムにログインしたい
**なぜなら**: 自身の権限に応じた機能に安全にアクセスするためだ

**受け入れ基準**:

- [ ] メールアドレスとパスワードを入力してログインできる
- [ ] ログイン成功後、ロールに応じたトップ画面に遷移する
- [ ] 認証失敗時にエラーメッセージが表示される
- [ ] 5 回連続失敗でアカウントが一時ロックされる
- [ ] 未ログイン状態でシステムにアクセスするとログイン画面にリダイレクトされる

**対応ユースケース**: UC-013

---

#### US-018: 得意先アカウントを新規登録する

**として**: 得意先
**したい**: 新規アカウントを作成して WEB ショップを利用開始したい
**なぜなら**: 花束を注文できるようになるためだ

**受け入れ基準**:

- [ ] メールアドレス・パスワード・氏名・連絡先を入力して登録できる
- [ ] 登録済みのメールアドレスの場合はエラーが表示される
- [ ] 登録後、自動的にログインされる

**対応ユースケース**: UC-013

---

#### US-019: 注文をキャンセルする

**として**: 受注スタッフ
**したい**: 得意先の依頼に基づいて注文をキャンセルしたい
**なぜなら**: 得意先の都合に合わせて柔軟に対応するためだ

**受け入れ基準**:

- [ ] 受注詳細画面から「キャンセル」を実行できる
- [ ] 「出荷準備中」以降のステータスではキャンセルできない
- [ ] キャンセル時に在庫引当が解除される
- [ ] キャンセル後、受注一覧に反映される

**対応ユースケース**: UC-003
