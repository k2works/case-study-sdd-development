# ユーザーストーリー

本書は、システムユースケースから開発可能な粒度へ分解したユーザーストーリーです。各ストーリーは受け入れ基準を持ち、ユースケースとの対応関係を明示します。

## フェーズ別マッピング

| フェーズ | ストーリー | 対応ユースケース |
| :--- | :--- | :--- |
| MVP | US-01, US-02, US-03, US-04 | UC-01, UC-02, UC-05 |
| フェーズ 2 | US-00, US-00B, US-05, US-06, US-09, US-09B, US-10 | UC-00, UC-00B, UC-06, UC-07, UC-08, UC-08B, UC-09 |
| フェーズ 3 | US-07, US-08 | UC-04, UC-03 |

## MVP

### US-01 商品を選んで注文内容を入力したい

**ストーリー**: 得意先として、花束を選び、届け日、届け先、メッセージを入力したい。なぜなら、記念日に合わせた注文を自分で完了したいからだ。

**対応ユースケース**: UC-01

**受け入れ基準**:

- 商品一覧から 1 つの商品を選択できる。
- 届け日、届け先、メッセージを入力できる。
- 必須項目が未入力の場合は確定前にエラー表示される。

### US-02 注文内容を確認して確定したい

**ストーリー**: 得意先として、入力した注文内容を確認してから確定したい。なぜなら、届け日や届け先の誤りを防ぎたいからだ。

**対応ユースケース**: UC-01

**受け入れ基準**:

- 入力内容の確認画面が表示される。
- 修正したい場合は入力画面へ戻れる。
- 確定すると受注が登録され、完了画面が表示される。

### US-03 受注一覧と詳細を確認したい

**ストーリー**: 受注スタッフとして、受注一覧と詳細を確認したい。なぜなら、顧客対応と後続業務に必要な情報をすぐ参照したいからだ。

**対応ユースケース**: UC-02

**受け入れ基準**:

- 受注一覧に主要項目が表示される。
- 一覧から任意の受注詳細を開ける。
- 対象受注がない場合は空状態が分かる。

### US-04 日別の在庫推移を確認したい

**ストーリー**: 仕入スタッフとして、日別の在庫推移を確認したい。なぜなら、不足リスクや廃棄リスクを見ながら発注判断したいからだ。

**対応ユースケース**: UC-05

**受け入れ基準**:

- 対象期間を指定して在庫推移を表示できる。
- 単品ごとの日別在庫予定数が確認できる。
- 不足見込みまたは廃棄注意の対象が識別表示される。

## フェーズ 2

### US-00 花束商品と花束構成を管理したい

**ストーリー**: 受注スタッフとして、販売する花束商品と花束構成を管理したい。なぜなら、受注可能な商品を最新状態に保ちたいからだ。

**対応ユースケース**: UC-00

**受け入れ基準**:

- 商品一覧から新規登録または編集を開始できる。
- 商品名、価格、販売状態、花束構成を入力できる。
- 保存後に受注画面で利用可能な状態が反映される。

### US-00B 花材と仕入条件を管理したい

**ストーリー**: 仕入スタッフとして、花材、品質維持日数、仕入条件を管理したい。なぜなら、在庫推移と発注判断の前提データを正しく保ちたいからだ。

**対応ユースケース**: UC-00B

**受け入れ基準**:

- 花材一覧から新規登録または編集を開始できる。
- 品質維持日数、購入単位、リードタイムを入力できる。
- 保存後に在庫推移と発注候補へ反映される。

### US-05 仕入先別に発注を確定したい

**ストーリー**: 仕入スタッフとして、在庫推移から必要な単品を仕入先別に発注したい。なぜなら、必要な花材を必要なタイミングで確保したいからだ。

**対応ユースケース**: UC-06

**受け入れ基準**:

- 発注対象候補から対象を選択できる。
- 仕入先別の発注内容が確認できる。
- 発注確定後に `送信待ち` または `送信済み` の状態が分かる。
- 数量 `0` または負数の発注は確定できない。
- 同一対象を重複確定しようとした場合は、確認メッセージまたは再送案内が表示される。

### US-06 入荷実績を登録して在庫へ反映したい

**ストーリー**: 仕入スタッフとして、入荷実績を登録したい。なぜなら、実際に使える在庫を正しく把握したいからだ。

**対応ユースケース**: UC-07

**受け入れ基準**:

- 対象発注に対して入荷数量を入力できる。
- 一部入荷と入荷完了を区別して保持できる。
- 登録後に在庫予定へ反映される。
- 入荷数量が `0` 未満または未入力の場合は登録できない。
- 発注数量を超える入荷数量は確認なしでは登録できない。

### US-09 出荷対象と必要花材を確認したい

**ストーリー**: フローリストとして、出荷対象と必要花材を確認したい。なぜなら、当日の結束作業を漏れなく進めたいからだ。

**対応ユースケース**: UC-08

**受け入れ基準**:

- 出荷日の対象受注一覧が表示される。
- 各受注に必要な花材を確認できる。
- 対象がない場合はその旨が表示される。

### US-09B 花束の結束完了を登録したい

**ストーリー**: フローリストとして、結束済みの花束を出荷準備完了として登録したい。なぜなら、受注スタッフへ安全に引き継ぎたいからだ。

**対応ユースケース**: UC-08B

**受け入れ基準**:

- 出荷対象から任意の対象を選んで結束完了を登録できる。
- 登録後に出荷状態が `出荷準備完了` へ更新される。
- 在庫不足または保留中の対象は結束完了登録できない。
- すでに `出荷準備完了` または `出荷済み` の対象は二重登録できない。

### US-10 出荷実績を確定したい

**ストーリー**: 受注スタッフとして、出荷実績を確定したい。なぜなら、出荷済みの受注を正しく管理したいからだ。

**対応ユースケース**: UC-09

**受け入れ基準**:

- `出荷準備完了` の対象だけを選択して確定できる。
- 確定後に出荷実績が保存される。
- 受注状態が出荷済みに更新される。
- すでに `出荷済み` の対象は二重確定できない。
- 対象が `0` 件の場合は確定操作できない。

## フェーズ 3

### US-07 過去の届け先をコピーして再注文したい

**ストーリー**: リピーターとして、過去の届け先をコピーしたい。なぜなら、同じ届け先へ素早く再注文したいからだ。

**対応ユースケース**: UC-04

**受け入れ基準**:

- 注文者のメールアドレスと電話番号で過去注文を照合できる。
- 過去の届け先一覧を表示できる。
- 任意の届け先を選択すると注文入力へ反映される。
- 履歴がない場合は新規入力へ誘導される。

### US-08 条件を満たす場合に届け日を変更したい

**ストーリー**: 受注スタッフとして、条件を満たす受注の届け日を変更したい。なぜなら、顧客要望に応えつつ業務上の無理を避けたいからだ。

**対応ユースケース**: UC-03

**受け入れ基準**:

- 対象受注に新しい届け日を入力できる。
- 在庫不足または出荷準備済みの場合は変更不可になる。
- 変更可能な場合は新しい届け日が保存される。
