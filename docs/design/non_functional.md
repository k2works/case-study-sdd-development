## 可用性要件

| 指標 | 目標 | 備考 |
| :--- | :--- | :--- |
| 稼働率 | 99.5%（月間ダウンタイム 3.6 時間以内） | 記念日前の夜間注文を考慮 |
| 計画停止 | 深夜 2:00〜4:00（月 1 回） | メンテナンスウィンドウ |
| 障害復旧目標（RTO） | 1 時間以内 | ECS タスク再起動 |
| データ復旧目標（RPO） | 1 時間以内 | RDS 自動バックアップ |

### 可用性の実現方法

- ECS Fargate: タスク数 2（Multi-AZ）+ ALB ヘルスチェック
- RDS: Multi-AZ 配置、自動フェイルオーバー
- CloudFront: 静的ファイル配信のキャッシュ

## セキュリティ要件

### 認証・認可

| 要件 | 実現方法 |
| :--- | :--- |
| 得意先認証 | メール + パスワード（Django 標準認証） |
| パスワードポリシー | 8 文字以上、英数字混在 |
| セッション管理 | Django Session（有効期間 24 時間） |
| スタッフ認証 | Django Admin 認証 |
| 認可 | ロールベース（customer / staff / admin） |

### データ保護

| 要件 | 実現方法 |
| :--- | :--- |
| 通信暗号化 | HTTPS（AWS ACM 証明書） |
| パスワード保存 | bcrypt ハッシュ（Django 標準） |
| 個人情報保護 | 得意先の氏名・住所・電話番号は暗号化通信のみ |
| CSRF 対策 | Django CSRF ミドルウェア |
| XSS 対策 | Django テンプレートの自動エスケープ |
| SQL インジェクション対策 | Django ORM（パラメータバインド） |
