# 非機能要件 - フレール・メモワール WEB ショップ

## 性能要件

| 項目 | 目標値 | 根拠 |
| :--- | :--- | :--- |
| 画面表示時間 | 3 秒以内 | 一般的な WEB アプリの許容範囲 |
| API レスポンス | 500ms 以内 | 在庫推移計算が最も重い処理 |
| 同時接続数 | 50 ユーザー | 個人顧客 + スタッフの同時利用想定 |
| 1 日あたり受注件数 | 100 件 | 初期の事業規模想定 |

## 可用性要件

| 項目 | 目標値 | 根拠 |
| :--- | :--- | :--- |
| 稼働率 | 99%（月間ダウンタイム 7.3 時間以内） | 小規模 WEB ショップとして十分 |
| 計画停止 | 月 1 回、深夜帯（2:00-5:00） | メンテナンスウィンドウ |
| データバックアップ | 日次 | RPO: 24 時間 |
| 復旧目標 | 4 時間以内 | RTO: 4 時間 |

## セキュリティ要件

| 項目 | 対策 |
| :--- | :--- |
| 通信暗号化 | HTTPS（TLS 1.2 以上） |
| 入力検証 | サーバーサイドバリデーション必須 |
| SQL インジェクション | パラメータバインド（ORM 使用） |
| XSS | 出力エスケープ |
| CSRF | トークン検証 |
| 認証 | セッションベース認証（得意先・スタッフ） |
| 個人情報保護 | 得意先の氏名・住所・電話番号を適切に管理 |

## 保守性要件

| 項目 | 目標値 | 根拠 |
| :--- | :--- | :--- |
| テストカバレッジ | 80% 以上 | 安全な変更のための安全網 |
| コード品質 | Lint エラー 0 件 | 一貫した品質維持 |
| デプロイ頻度 | 週 1 回以上 | 継続的デリバリー |
| デプロイ所要時間 | 15 分以内 | CI/CD パイプライン自動化 |

## 拡張性要件

| 項目 | 方針 |
| :--- | :--- |
| 商品数増加 | マスタデータの追加で対応（コード変更不要） |
| 受注件数増加 | DB インデックスの最適化で対応 |
| 新機能追加 | ヘキサゴナルアーキテクチャにより、ドメイン層に影響を与えずに拡張可能 |

## ユーザビリティ要件

| 項目 | 目標 |
| :--- | :--- |
| 対応ブラウザ | Chrome / Safari / Edge（最新 2 バージョン） |
| レスポンシブ対応 | PC 優先（スマートフォンは将来対応） |
| 操作ステップ | 注文完了まで 4 ステップ以内 |
| エラーメッセージ | 日本語で具体的な原因と対処法を表示 |
