## バージョン管理方針

### 基本方針

| 項目 | 方針 |
|:---|:---|
| LTS 優先 | 可能な限り LTS（Long Term Support）バージョンを選択する |
| メジャーバージョン更新 | セキュリティリスク・EOL を考慮し、四半期ごとに評価する |
| マイナー・パッチ更新 | Dependabot による自動 PR で月次更新を検討する |
| 破壊的変更 | メジャーバージョン更新前に影響調査とテスト実行を必須とする |

### EOL 管理

| 技術 | 現行バージョン | EOL | 次回評価時期 |
|:---|:---|:---|:---|
| Java | 21 LTS | 2031-09 | 不要（長期サポート） |
| Spring Boot | 3.x | 各リリースから 12 ヶ月 | 半期ごと |
| React | 19.x | 未定 | メジャーバージョンリリース後 |
| PostgreSQL | 15 | 2027-11 | 2027 Q1 |
| Node.js | 22 LTS | 2027-04 | 2027 Q1（次期 LTS 評価） |

### セキュリティ更新ポリシー

- **Critical**: 検知後 24 時間以内に対応
- **High**: 1 週間以内に対応
- **Medium**: 次回定期メンテナンスで対応
- **Low**: 次回メジャー更新時に対応
