## 補足

今回のアーキテクチャ設計では、以下を基本方針としています。

- 小規模な XP チームでも運用しやすいモジュラーモノリスを採用する
- バックエンドは 4 層レイヤードを基本に、外部境界のみポート / アダプターを採用する
- フロントエンドは顧客向けとスタッフ向けを単一 Web アプリで構成する
- インフラはコンテナ化とマネージドサービスを前提に、過剰な分散構成を避ける
