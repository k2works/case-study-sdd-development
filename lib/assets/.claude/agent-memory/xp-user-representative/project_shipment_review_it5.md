IT5 出荷管理機能のレビューを実施。主要な発見:

- ship_all が個別トランザクションで部分的に処理される問題（高優先）
- 出荷処理後に出荷日フィルタがリセットされるUX問題（高優先）
- 配送先情報・受注数量の一覧表示が不足
- コントローラで delivery_date = shipping_date + 1.day をハードコードしており Order#shipping_date と二重定義
- 在庫消費タイミングが出荷時だが実業務では結束時に消費される乖離あり（将来検討）
- 出荷取消機能・出荷履歴参照がスコープ外だが運用上必要

**Why:** S11/S12 の受入条件は概ね満たすが、現場運用で混乱を招く実装上の問題がある
**How to apply:** 高優先の3件（トランザクション問題、フィルタ引継ぎ、受注数量表示）は本イテレーション内で対応推奨
